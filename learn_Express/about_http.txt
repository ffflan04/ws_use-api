TCP/IPの４階層モデル

HTTP のベースは、TCP/IP というネットワークプロトコル

アプリケーション層 => HTTP：アプリケーションごとの通信プロトコルを決定する層
トランスポート層 => TCP：データ転送の抜け漏れをチェックする層
    ポート番号を使って、接続先とコネクションをはることを行ってるのが、トランスポート層。
インターネット層 => IP：データを送るための層
ネットワークインターフェース層 => Ethernet：LANケーブルなどの物理的な層

HTTPメッセージ概要
httpリクエスト、httpレスポンスにて送受信される、Httpメッセージ

curlコマンド = 
    ウェブページのHTMLコンテンツを取得
    APIに対してHTTPリクエストを送信し、データの取得や更新、削除を行う
    様々なプロトコルを使用した、ファイルのアップ/ダウンロード,
    Basic認証、Digest認証、トークン認証など、さまざまな認証方法に対応したリクエストを送信
    プロキシを経由したリクエスト => curl -X
など、一見、最強そうですね。
curl http:// -v で、httpレスポンスの詳細を見れる。


Httpリクエスト
・ヘッダー(メタデータ)
    accept：要求するデータ形式( JSONなど )
    Host：リクエスト先のホスト名
    User-Agent：リクエスト元情報
・ボディ(実データ)
    引数みたいな感じ？
    作成したいリソースの情報など。
    (PostやPutの時に使う。)

Httpレスポンス
・ヘッダー(メタデータ)
    content-type：返ってくるデータ形式( JSONなど )
・ボディ(実データ)
    サーバーが返す実際のデータ( HTMLやJSON )

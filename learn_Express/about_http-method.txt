/* ------ Httpメソッドについて ------ */

Httpはステートレスであれ。
・サーバーがクライアントの状態を管理せずに通信すべし。
・ステートフルな通信は、簡潔であるが、接続先が多くなると管理しきれません。
・ステートレスな通信は、データ量が多く冗長であるが、シンプル。
・ステートレスで通信エラーが起きると、リクエストが重複する可能性がある。


主要なHttpメソッド
・Httpメソッドは計８つあるが、主要メソッドは６つ
・CRUDは４つのメソッドで実装することが多い。
・GET => Read -> リソースの取得
・POST => Create -> リソースの作成( 万能なメソッド )
・PUT => Update -> リソースの更新
・DELETE => Delete -> リソースの削除


URI設計例

GET：/api/v1/users => ユーザーリストの取得
GET：/api/v1/users/123 => ユーザー情報の取得
POST：/api/v1/users => 新規ユーザーの作成
PUT：/api/v1/users/123 => ユーザー情報の更新
DELETE：/api/v1/users/123 => ユーザーの削除
GET：/api/v1/search?q=username => ユーザー検索結果の取得
